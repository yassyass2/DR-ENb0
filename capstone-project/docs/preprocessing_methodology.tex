\begin{abstract}
This document describes the current reproducible pipeline for five-class
diabetic retinopathy grading on the APTOS~2019 fundus image dataset. The
pipeline intentionally uses a narrow baseline scope: APTOS~2019 only,
green-channel preprocessing, an ImageNet-pretrained EfficientNetB0 classifier,
class-weighted baseline training, a separate optional SMOTE experiment, and
Grad-CAM visualisation. The purpose of this version is to stabilise the data,
model, metric, and explainability contracts before extending the project with
additional datasets or explanation methods.
\end{abstract}

\section{Dataset}
\label{sec:dataset}

The project uses the APTOS~2019 Blindness Detection dataset. Each image is
labelled on the five-point diabetic retinopathy (DR) severity scale:
\emph{0~--~no DR}, \emph{1~--~mild}, \emph{2~--~moderate},
\emph{3~--~severe}, and \emph{4~--~proliferative DR}. The current run uses
\num{2930} training images, \num{366} validation images, and \num{366} held-out
test images.

The input photographs vary in resolution, aspect ratio, illumination, and
camera framing. The preprocessing pipeline therefore standardises each image
before training while preserving the natural validation and test distributions.

\section{Preprocessing Pipeline}
\label{sec:pipeline}

Images are converted to RGB, black borders are cropped using a threshold of 10,
the fundus is resized to \(224 \times 224\) with aspect-ratio-preserving zero
padding, the green channel is extracted, CLAHE is applied with clipLimit 2.0
and an \(8 \times 8\) tile grid, median filtering is applied with a \(3 \times 3\)
kernel, and pixel values are normalized to \texttt{float32} \([0,1]\).

The resulting saved array contract is:

\begin{center}
\begin{tabular}{lll}
\toprule
Split & Image tensor & Label tensor \\
\midrule
Train & \((2930, 224, 224, 1)\), \texttt{float32}, \([0,1]\) & \((2930,)\) \\
Validation & \((366, 224, 224, 1)\), \texttt{float32}, \([0,1]\) & \((366,)\) \\
Test & \((366, 224, 224, 1)\), \texttt{float32}, \([0,1]\) & \((366,)\) \\
\bottomrule
\end{tabular}
\end{center}

\subsection{Green Channel and CLAHE}
\label{sec:green-clahe}

The green channel is used because retinal vessels, haemorrhages, and several
small lesion patterns generally have stronger contrast there than in the red
or blue channels. CLAHE is then applied directly to the green channel to
enhance local contrast without applying a global histogram transform to the
full image.

\subsection{Class Imbalance Handling}
\label{sec:imbalance}

The canonical baseline does not alter the validation or test distribution.
During baseline training, class imbalance is handled with class weights derived
from the training labels. SMOTE is treated as a separate experiment rather than
as part of the default baseline. In the SMOTE experiment, only the training
split is oversampled; validation and test arrays remain unchanged.

\section{Model Contract}
\label{sec:model}

The model accepts one-channel preprocessed green images in \([0,1]\). Inside
the model, the channel is scaled to the EfficientNetB0 input range and repeated
to three channels for compatibility with ImageNet-pretrained EfficientNetB0.
This keeps the external data contract simple while allowing the backbone to use
pretrained weights.

The baseline model is selected using validation quadratic weighted kappa (QWK),
with early stopping and learning-rate reduction also monitored through the
validation run.

\section{Evaluation}
\label{sec:evaluation}

Classification performance is evaluated with QWK as the primary metric,
supplemented by accuracy, macro sensitivity, macro specificity, macro ROC-AUC,
per-class sensitivity, per-class specificity, and a confusion matrix.

\section{Baseline Results}
\label{sec:results}

The canonical EfficientNetB0 baseline reached a validation QWK of \(0.8648\) at
epoch 17. Early stopping stopped training at epoch 25 and restored the epoch 17
weights. On the held-out test split, the model achieved the results shown in
Table~\ref{tab:test-results}.

\begin{table}[t]
  \centering
  \caption{Held-out test performance of the canonical EfficientNetB0 baseline.}
  \label{tab:test-results}
  \begin{tabular}{lr}
    \toprule
    Metric & Value \\
    \midrule
    Quadratic weighted kappa & 0.8655 \\
    Accuracy & 0.7842 \\
    Macro sensitivity & 0.6431 \\
    Macro specificity & 0.9492 \\
    Macro ROC-AUC & 0.9204 \\

The main weakness is minority-grade recall. Sensitivity is strongest for grade
0 (\(0.9598\)) and lower for grade 1 (\(0.5333\)) and grade 4 (\(0.4545\)).
A separate SMOTE ablation reached similar QWK (\(0.8629\)) but lower macro
sensitivity (\(0.5997\)), mainly because grade 1 sensitivity dropped to
\(0.2667\). The class-weighted baseline therefore remains the selected result.

\section{Explainability}
\label{sec:explainability}

Grad-CAM is generated from the trained EfficientNetB0 model by exposing the
final convolutional activation and replaying the exact classifier head. The
implementation is validated by checking that Grad-CAM model predictions match
the original model predictions before heatmaps are generated.

The current explainability output should be interpreted as qualitative visual
evidence only. Several heatmaps highlight retinal structures plausibly related
to disease severity, but some also focus on the optic disc or image borders.
Those cases are treated as failure-mode evidence and should not be presented as
quantitative lesion-level validation.

\section{Reproducibility}
\label{sec:reproducibility}

All generated artifacts for the current baseline are stored under the project
\texttt{artifacts/} directory. The main files are:

\begin{itemize}
  \item \texttt{test\_report.json}: held-out test metrics and confusion matrix.
  \item \texttt{figures/confusion\_matrix.png}: count and row-normalised confusion matrices.
  \item \texttt{figures/per\_class\_metrics.png}: sensitivity, specificity, and AUC per grade.
  \item \texttt{figures/training\_curves.png}: loss, accuracy, and validation QWK curves.
  \item \texttt{figures/gradcam\_summary.png}: one Grad-CAM example per grade.
  \item \texttt{figures/gradcam\_grade\_0.png} through \texttt{gradcam\_grade\_4.png}: per-grade Grad-CAM grids.
\end{itemize}
